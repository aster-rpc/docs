## Define a service

<LanguageTabs defaultValue="python">
<TabItem value="python" label="Python">

Create `hello_service.py`:

```python
from dataclasses import dataclass
from aster.decorators import service, rpc


@dataclass
class HelloRequest:
    name: str = ""


@dataclass
class HelloResponse:
    message: str = ""


@service
class HelloService:
    @rpc
    async def say_hello(self, req: HelloRequest) -> HelloResponse:
        return HelloResponse(message=f"Hello, {req.name}!")
```

Three decorators: `@dataclass` (standard Python), `@service` (marks the class as an Aster service), `@rpc` (marks a method as a callable endpoint). No schema files, no code generation, no base classes.

</TabItem>
<TabItem value="typescript" label="TypeScript">

Create `types.ts`:

```typescript
import { WireType } from '@aster-rpc/aster';

@WireType("hello/HelloRequest")
export class HelloRequest {
  name = "";
  constructor(init?: Partial<HelloRequest>) {
    if (init) Object.assign(this, init);
  }
}

@WireType("hello/HelloResponse")
export class HelloResponse {
  message = "";
  constructor(init?: Partial<HelloResponse>) {
    if (init) Object.assign(this, init);
  }
}
```

Create `service.ts`:

```typescript
import { Service, Rpc } from '@aster-rpc/aster';
import { HelloRequest, HelloResponse } from './types.js';

@Service({ name: "HelloService", version: 1 })
export class HelloService {
  @Rpc()
  async sayHello(req: HelloRequest): Promise<HelloResponse> {
    return new HelloResponse({
      message: `Hello, ${req.name}!`,
    });
  }
}
```

Three decorators: `@WireType` (stable wire identity), `@Service` (marks the class), `@Rpc` (marks a method). No schema files, no code generation, no base classes.

</TabItem>
</LanguageTabs>

## Run a producer

A producer is the node that hosts the service.

<LanguageTabs defaultValue="python">
<TabItem value="python" label="Python">

Create `producer.py`:

```python
import asyncio
from hello_service import HelloService
from aster import AsterServer


async def main():
    async with AsterServer(services=[HelloService()]) as srv:
        print("Producer ready at:", srv.address)
        await srv.serve()


asyncio.run(main())
```

```bash
python producer.py
```

</TabItem>
<TabItem value="typescript" label="TypeScript">

Create `producer.ts`:

```typescript
import { AsterServer } from '@aster-rpc/aster';
import { HelloService } from './service.js';

const server = new AsterServer({
  services: [new HelloService()],
});

await server.start();
console.log("Producer ready at:", server.address);
await server.serve();
```

```bash
bun run producer.ts
```

</TabItem>
</LanguageTabs>

In dev mode (no `ASTER_*` environment variables set), the server:

- Generates an ephemeral root key and node identity (no files needed).
- Opens the consumer gate so consumers can connect without credentials.
- Serves RPC, blobs, docs, and gossip on a single endpoint.
- Prints an `aster1...` address for consumers to connect to.

## Run a consumer

<LanguageTabs defaultValue="python">
<TabItem value="python" label="Python">

Create `consumer.py`:

```python
import asyncio
from hello_service import HelloService, HelloRequest
from aster import AsterClient


async def main():
    async with AsterClient() as c:
        hello = await c.client(HelloService)
        resp = await hello.say_hello(HelloRequest(name="World"))
        print(resp.message)  # Hello, World!


asyncio.run(main())
```

Run it, passing the producer's address:

```bash
ASTER_ENDPOINT_ADDR=<paste from producer output> python consumer.py
```

`AsterClient` reads `ASTER_ENDPOINT_ADDR` from the environment, connects to the producer's admission endpoint (to discover available services), then opens an RPC connection. `c.client(HelloService)` returns a typed client stub -- call methods on it like regular async functions.

</TabItem>
<TabItem value="typescript" label="TypeScript">

Create `consumer.ts`:

```typescript
import { AsterClientWrapper } from '@aster-rpc/aster';
import { HelloService } from './service.js';
import { HelloRequest } from './types.js';

const client = new AsterClientWrapper({
  address: process.env.ASTER_ENDPOINT_ADDR!,
});

await client.connect();
const hello = await client.proxy("HelloService");
const resp = await hello.sayHello({ name: "World" });
console.log(resp.message); // Hello, World!
await client.close();
```

Run it, passing the producer's address:

```bash
ASTER_ENDPOINT_ADDR=<paste from producer output> bun run consumer.ts
```

</TabItem>
</LanguageTabs>

## Dev mode vs production

The quickstart above runs in **dev mode** -- ephemeral keys, open gates, no credential files. Everything works out of the box for local development.

In **production**, you use the CLI to manage trust and enroll nodes:

```bash
# Operator's machine (one time):
aster keygen root
#   -> ~/.aster/root.key (private -- keep secret)
#   -> ~/.aster/root.pub (public -- share with nodes)

# Enroll this node as a producer:
aster enroll node --role producer --name hello-producer
#   -> generates node key + signs credential -> writes .aster-identity

# Enroll the same node as a consumer:
aster enroll node --role consumer --name hello-consumer
#   -> adds a consumer peer entry to the same .aster-identity
```

Then deploy the `.aster-identity` file alongside your code:

<LanguageTabs defaultValue="python">
<TabItem value="python" label="Python">

```python
# producer.py -- loads node key + producer credential from .aster-identity
async with AsterServer(services=[HelloService()], peer="hello-producer") as srv:
    print(srv.address)
    await srv.serve()

# consumer.py -- loads consumer credential from .aster-identity
async with AsterClient(peer="hello-consumer") as c:
    hello = await c.client(HelloService)
    resp = await hello.say_hello(HelloRequest(name="World"))
```

```bash
# Consumer still needs to know where the producer is:
ASTER_ENDPOINT_ADDR=<producer addr> python consumer.py
```

</TabItem>
<TabItem value="typescript" label="TypeScript">

```typescript
// producer.ts -- loads node key + producer credential from .aster-identity
const server = new AsterServer({
  services: [new HelloService()],
  config: { peer: "hello-producer" },
});
await server.start();
console.log(server.address);
await server.serve();

// consumer.ts -- loads consumer credential from .aster-identity
const client = new AsterClientWrapper({
  config: { peer: "hello-consumer" },
  address: process.env.ASTER_ENDPOINT_ADDR!,
});
await client.connect();
```

```bash
# Consumer still needs to know where the producer is:
ASTER_ENDPOINT_ADDR=<producer addr> bun run consumer.ts
```

</TabItem>
</LanguageTabs>

Here we enroll the same node as both producer and consumer. In practice, producers and consumers are often separate nodes -- see *Understanding the .aster-identity file (coming soon)*.
